## Root Cause

**Bug: Invalid SQL generated by `sql ANY()` in `question-pool.ts`**

The `getQuestionsFromPool` function marked questions as used after fetching them:
```typescript
// BROKEN — generates: WHERE id = ANY(($1, $2)) — invalid SQL
.where(sql`${questionPool.id} = ANY(${ids})`)
```

Drizzle ORM serializes arrays as `($1, $2)` tuple syntax in this context, but PostgreSQL `ANY()` requires an array: `ANY(ARRAY[$1, $2])`. This caused the UPDATE to throw on **every** question fetch attempt.

The error was silently swallowed in `getAlwaysOnQuestions`:
```typescript
getQuestionsFromPool(cat, 'medium', 2).catch(() => [])  // swallowed!
```

With all 10 category fetches returning `[]`, the total question count was 0, triggering: `"No questions available — seed the question pool first"`.

**The question pool itself was fully seeded** — 51–121 questions per category/difficulty (3,200+ total). The error was a query bug, not a missing data problem.

---

## Fixes Applied

### Fix 1: `src/lib/question-pool.ts` — Use `inArray()` instead of `sql ANY()`
```typescript
// BEFORE
import { eq, and, asc, sql } from 'drizzle-orm'
.where(sql`${questionPool.id} = ANY(${ids})`)

// AFTER
import { eq, and, asc, inArray, sql } from 'drizzle-orm'
.where(inArray(questionPool.id, ids))
```
Commit: `15430f9`

### Fix 2: `src/app/api/games/live/current/route.ts` — Better timing for bar display
```typescript
// BEFORE
const ALWAYS_ON_TIME_PER_QUESTION = 30 // seconds
const LEADERBOARD_DURATION_MS = 5000   // 5 seconds

// AFTER
const ALWAYS_ON_TIME_PER_QUESTION = 15 // seconds — question shown, no answer
const LEADERBOARD_DURATION_MS = 8000   // 8 seconds — answer + leaderboard visible
```
30s question was too long for bar/restaurant TV display. 5s answer reveal was too short to read. New: 15s question → 8s answer = 23s per question cycle.
Commit: `1fb3a2b`

### Fix 3: Added `ANTHROPIC_API_KEY` to Vercel (Production + Development)
The key was missing from Vercel env entirely, meaning the fallback question generation (when pool is low) would fail silently.

## How TV Auto-Play Works Now

1. Browser navigates to `https://aitriviaarena.com/tv`
2. Component mounts → `useEffect` fires immediately → starts polling `GET /api/games/live/current` every 1.5 seconds
3. Server checks for today's always-on game in DB. If none exists, creates one by pulling 2 questions from each of 10 categories (20 total)
4. Response: `{ status: 'active', question: {...}, timePerQuestion: 15, ... }`
5. TV page renders `<QuestionScreen>` with countdown timer (15s)
6. After 15s, server detects time elapsed → transitions to `showing_leaderboard` (answer revealed for 8s)
7. After 8s, server advances to next question, loops forever
8. No user interaction ever required — works with Display Cast, Chromecast, AirPlay
